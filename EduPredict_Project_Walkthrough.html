🧠 Machine Learning Pipeline

Dataset

UCI Student Performance Data

Source: UCI Machine Learning Repository — Student Performance (Mathematics course)

Records: 395 students  |  Target: Final grade G3 mapped to 3 classes:

  • Class 0 — Poor (G3 < 10)
  • Class 1 — Average (10 ≤ G3 < 14)
  • Class 2 — Good (G3 ≥ 14)

Features Used (7 total)

FeatureDescriptionType
attendance100 - absences (engineered)Numeric
internal_avg(G1 + G2) / 2 (engineered)Numeric
studytimeWeekly study time (1–4 scale)Numeric
backlogsNumber of past class failuresNumeric
schoolsupExtra educational supportCategorical → 0/1
activitiesExtracurricular activitiesCategorical → 0/1
higherWants to pursue higher educationCategorical → 0/1

Training Scripts

1. student_performance_model.py — Base Model

  • Trains both Random Forest (300 trees, max_depth=12) and XGBoost (300 trees, lr=0.05)
  • 80/20 stratified split + StandardScaler normalization
  • Evaluates with accuracy, F1-score, 5-fold cross-validation
  • Auto-saves the best-performing model

2. student_performance_tuned_model.py — Tuned Model

  • Adds GridSearchCV hyperparameter tuning for both models
  • RF grid: n_estimators ∈ {200, 300, 400}, max_depth ∈ {8, 12, 16}, min_samples_split, min_samples_leaf
  • XGBoost grid: n_estimators, learning_rate, max_depth, subsample, colsample_bytree
  • 5-fold CV with f1_weighted scoring
  • Saves final champion model as final_student_model.pkl

🔌 Backend API

Routes

RouteMethodDescription
/GETHealth check — returns "API is running"
/predictPOSTAccepts student features as JSON, returns prediction

Sample API Response

{ "prediction": 2, "performance": "Good Performance", "confidence": 0.87, "feature_importance": { "attendance": 0.12, "internal_avg": 0.45, "studytime": 0.08, "backlogs": 0.15, "schoolsup": 0.03, "activities": 0.05, "higher": 0.12 } }

Key details: Uses CORS for cross-origin requests. Loads model & scaler at startup. Scales input features before prediction. Returns class label, human-readable performance tier, confidence score (max probability), and per-feature importance scores.

🎨 Frontend Application

The frontend is a single-page React application with client-side routing managed by a state variable. Protected behind a login gate — users authenticate before accessing the dashboard.

Key Features

🔐

Authentication Gate

Email-based login page with dark/light mode toggle. User session in state. Profile dropdown with sign-out.

📊

2-Step Prediction Wizard

Step 1: Academic stats (attendance, avg, failures). Step 2: Study habits & support. Auto-fill & reset buttons.

📈

Rich Result Display

Animated score counter, 3 metric cards, projected trajectory SVG chart, feature impact grid, AI advice panel.

📋

Prediction History

Stored in localStorage (up to 20 entries). Notification bell in header. Dedicated browsing page.

👨‍🎓

Students Directory

Full student listing with detailed profile modals and radar chart visualizations.

🧭

Responsive Navigation

Collapsible sidebar (icon/label modes). Hamburger menu on mobile. Animated page transitions.

🌙

Dark / Light Mode

Full theme toggle with smooth transitions. Applied globally via document class.

✨

Premium UI Polish

Skeleton loaders, toast notifications, micro-animations, gradient orbs, glassmorphism, spring physics.

🚀 How to Run

Backend

cd student-performance-api
pip install flask flask-cors numpy joblib scikit-learn
python app.py
# → API running at http://localhost:5000

Frontend

cd student-performance-frontend
npm install
npm run dev
# → App running at http://localhost:5173

Train a New Model (optional)

python student_performance_tuned_model.py
# → Saves final_student_model.pkl + scaler.pkl
