🧠 Machine Learning Pipeline

Dataset

UCI Student Performance Data

Source: UCI Machine Learning Repository — Student Performance (Mathematics course)

Records: 395 students  |  Target: Final grade G3 mapped to 3 classes:

  • Class 0 — Poor (G3 < 10)
  • Class 1 — Average (10 ≤ G3 < 14)
  • Class 2 — Good (G3 ≥ 14)

Features Used (7 total)

FeatureDescriptionType
attendance100 - absences (engineered)Numeric
internal_avg(G1 + G2) / 2 (engineered)Numeric
studytimeWeekly study time (1–4 scale)Numeric
backlogsNumber of past class failuresNumeric
schoolsupExtra educational supportCategorical → 0/1
activitiesExtracurricular activitiesCategorical → 0/1
higherWants to pursue higher educationCategorical → 0/1

Training Scripts

1. student_performance_model.py — Base Model

  • Trains both Random Forest (300 trees, max_depth=12) and XGBoost (300 trees, lr=0.05)
  • 80/20 stratified split + StandardScaler normalization
  • Evaluates with accuracy, F1-score, 5-fold cross-validation
  • Auto-saves the best-performing model

2. student_performance_tuned_model.py — Tuned Model

  • Adds GridSearchCV hyperparameter tuning for both models
  • RF grid: n_estimators ∈ {200, 300, 400}, max_depth ∈ {8, 12, 16}, min_samples_split, min_samples_leaf
  • XGBoost grid: n_estimators, learning_rate, max_depth, subsample, colsample_bytree
  • 5-fold CV with f1_weighted scoring
  • Saves final champion model as final_student_model.pkl

🔌 Backend API

Routes

RouteMethodDescription
/GETHealth check — returns "API is running"
/predictPOSTAccepts student features as JSON, returns prediction

Sample API Response

{ "prediction": 2, "performance": "Good Performance", "confidence": 0.87, "feature_importance": { "attendance": 0.12, "internal_avg": 0.45, "studytime": 0.08, "backlogs": 0.15, "schoolsup": 0.03, "activities": 0.05, "higher": 0.12 } }

Key details: Uses CORS for cross-origin requests. Loads model & scaler at startup. Scales input features before prediction. Returns class label, human-readable performance tier, confidence score (max probability), and per-feature importance scores.

🎨 Frontend Application

The frontend is a single-page React application with client-side routing managed by a state variable. Protected behind a login gate — users authenticate before accessing the dashboard. Features a cinematic loading screen with orbital animations after login, and a global floating AI chatbot available on every page.

🤖 AI Chatbot New

Overview

A globally available floating chatbot rendered on every authenticated page. Powered by the Groq API using the LLaMA 3.3 70B Versatile model. Implements Retrieval-Augmented Generation (RAG) by injecting student directory data from localStorage into the system prompt.

Key Capabilities

  • Career Roadmap Generation — Step-by-step guides based on target roles or engineering branches
  • Student Profile Analysis — Evaluates CGPA, internships, projects, and certifications against placement criteria
  • Job Readiness Assessment — Rates students across 4 pillars: CGPA, Internships, Projects, Certifications
  • RAG Context — Auto-retrieves top student profiles from the local directory to ground AI responses
  • Markdown Rendering — Parses bold text and bullet points inline for formatted chat responses
  • Expand/Collapse — Togglable between compact (380px) and expanded (800px) modes
  • Animated Toggle — Floating button with rotation animation and tooltip on hover

🚀 Loading Screen New

Post-Login Cinematic Experience

  • 5 sequential status messages with fade-in transitions (e.g., "Initiating quantum handshake…", "Calibrating reality matrix…")
  • Orbital animation — Nested counter-rotating rings around a pulsing Loader2 icon
  • Progress bar with shimmer effect (4.2s ease-in-out fill)
  • Floating particles — 5 animated dots with sinusoidal movement
  • Completion state — CheckCircle2 icon with spring entrance, "Ready!" message
  • Exit animation — Full-screen slide-up using cubic-bezier [0.76, 0, 0.24, 1]
  • Background gradient orbs with scale + opacity pulsing

👥 Students Directory

Architecture & Data

  • 350+ procedurally generated students — 70 per branch (CSE, IT, EXTC, Mechanical, Civil)
  • Realistic Indian names, enrollment numbers, branch-specific skills and certifications
  • CGPA distribution: 80% between 6.0–8.5, 20% high-performers 8.5–10.0
  • Persistent localStorage — Directory and custom profiles survive page reloads
  • Legacy profile migration — Auto-migrates older customStudentProfiles format

Features

  • Search — Filter by name, enrollment number, or skills in real-time
  • Branch Filters — Tab bar with per-branch student counts
  • Sorting — Default, Alphabetical (A-Z), CGPA (High to Low)
  • Bookmarks — Star/unstar any student; dedicated "Bookmarked" filter tab with toast feedback
  • Add Profile Modal — Create custom student profiles with name, enrollment, branch, CGPA, email, skills, certifications, attendance, projects, and internships
  • Student Profile Modal — 23KB component with detailed view, radar chart, recruiter tools
  • CGPA Color Coding — Emerald (≥9.0), Sky (≥7.5), Amber (≥6.5), Slate (below)
  • Lazy Loading — Load 24 students at a time with "Load More" pagination
  • Skeleton Loading — Animated placeholder cards during initial 600ms load

🚀 How to Run

Backend

cd student-performance-api
pip install flask flask-cors numpy joblib scikit-learn
python app.py
# → API running at http://localhost:5000

Frontend

cd student-performance-frontend
npm install
npm run dev
# → App running at http://localhost:5173

AI Chatbot Setup (optional)

# Create .env.local in student-performance-frontend/
VITE_GROQ_API_KEY=your_groq_api_key_here

# Get a free key at https://console.groq.com

Train a New Model (optional)

python student_performance_tuned_model.py
# → Saves final_student_model.pkl + scaler.pkl
